# MOTHER.md — Operating Rules

## RULE 1: DISCUSS BEFORE EXECUTE

NEVER execute changes (edits, commits, pushes) without discussing them with the founder first.

The process is:
1. Present the finding
2. State your recommendation and reasoning
3. WAIT for founder approval or direction
4. ONLY THEN execute

This applies to ALL audit items, code changes, content changes, and structural decisions. No exceptions.

## RULE 2: ONE AT A TIME

Go through items one by one. Present each item, discuss, get a decision, then move to the next. Do not batch or auto-advance.

## RULE 3: READ THIS FILE

Re-read this file before starting any new work session or processing any audit item.
